# 현업 실무를 위한 파이썬 UV 가상환경 구축 매뉴얼

**목적:** 아나콘다(Anaconda)보다 10배 빠른 최신 파이썬 패키지 관리자 `uv`를 활용하여, 프로젝트별로 독립된 데이터 분석 환경을 구축하고 GitHub과 연동한다.

### 1단계: 내 컴퓨터(로컬)에 UV 엔진 장착하기 (최초 1회만)

내 PC 본체에 `uv`라는 프로그램 자체를 설치하고, 어디서나 실행할 수 있도록 명령어 경로를 뚫어주는 단계입니다.

1. **Windows 시작 버튼 우클릭** ➔ **PowerShell(관리자)** 실행
2. 아래 코드를 정확히 복사해서 붙여넣고 **Enter** (uv 설치 진행)PowerShell
    
    ```
    powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
    ```
    
3. **체크 및 확인:** 열려 있는 터미널이나 Git Bash를 **완전히 종료한 후 다시 켜서** 아래 명령어를 입력합니다. 버전 숫자가 나오면 성공입니다.Bash
    
    ```
    uv --version
    ```
    

*(※ 만약 명령어를 못 찾겠다고 나오면, 시스템에 경로가 반영되도록 컴퓨터를 한 번 재부팅해 주세요.)*

### 2단계: 프로젝트 가상환경 방 만들기 (새 프로젝트 시작할 때마다)

1. 회사나 프로젝트 진행사에서 제공한 **zip 파일 압축 풀기**
2. 압축을 푼 폴더(`pyproject.toml` 파일이 존재하는 폴더) 안에서 **Git Bash(또는 터미널) 열기**
3. 아래 명령어 입력 (프로젝트에 지정된 파이썬 버전을 uv가 자동으로 감지하여 내부에 독립된 가상환경 `.venv` 폴더를 자동 생성하고 패키지까지 한 번에 맞춰줍니다)Bash
    
    ```
    uv sync
    ```
    

### 3단계: 작업 시작하기 (공부/개발 시작)

가상환경을 번거롭게 켜고 끄는(`activate`/`deactivate`) 과정 없이, `uv run` 명령어로 안전하고 독립된 주피터 랩을 구동합니다.

1. 프로젝트 폴더 터미널에서 아래 명령어 입력:Bash
    
    ```
    uv run jupyter lab
    ```
    
2. 자동으로 열리는 웹 브라우저 주피터 랩 환경에서 데이터 분석을 진행합니다.

### 4단계: 깃허브 올리기 및 삽질 방지

내가 짠 코드는 GitHub 원격 창고에 올리되, 수 기가바이트(GB)에 달하는 실제 라이브러리 설치 파일(`.venv` 폴더)은 업로드 대상에서 반드시 제외해야 합니다.

**① 깃허브 업로드 대상에서 가상환경 폴더 제외하기**
폴더 내에 `.gitignore` 파일이 없다면, 터미널(Git Bash)에 아래 명령어를 입력하여 간단하게 가상환경 제외 설정을 추가합니다. (메모장 확장자 실수 방지)

Bash

```
echo ".venv/" >> .gitignore
```

*(기존에 파일이 있다면 파일 맨 아랫줄에 `.venv/`가 잘 들어가 있는지 확인만 하시면 됩니다.)*

**② 코드 올리기 명령어 순서**
Git Bash에서 아래 순서대로 입력하면 안전하게 원격 창고로 업로드됩니다.

Bash

```
git add .
git commit -m "feat: 데이터 분석 환경 세팅 완료"
git push origin main
```
